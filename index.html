About ASSURE-AI

AI-assisted software engineering is rapidly changing how software is implemented, reviewed, tested, repaired, and maintained. While AI coding assistants and agentic development tools can improve productivity, their outputs may also introduce vulnerabilities, generate incomplete fixes, produce misleading security explanations, or create unwarranted confidence in insecure code.

ASSURE-AI focuses on evidence-based methods for assessing AI-generated and AI-modified software, including secure code generation, vulnerability detection, security testing, program analysis, patch validation, compliance reasoning, benchmark design, and human review.

Motivation

Productivity gains from AI-assisted development do not, by themselves, establish that the resulting software is secure or ready for deployment. ASSURE-AI addresses this gap by focusing on security claims that are supported by concrete evidence, such as static and dynamic analysis, security testing, human review, compliance mappings, formal reasoning, repository history, and reproducible benchmarks.

A key concern is false assurance: situations in which an AI tool, analysis pipeline, benchmark result, or explanation gives developers unwarranted confidence that software is secure.

Topics of Interest

We welcome submissions on topics including, but not limited to:

Security assurance for AI-generated and AI-modified software
Secure code generation and AI-assisted vulnerability detection
Security testing, program analysis, and patch validation
Evidence models for supporting security claims
False assurance, overconfident explanations, and developer overreliance
Compliance reasoning, auditability, and traceability
Benchmarks, datasets, empirical studies, and evaluation methods
Human-in-the-loop review and developer trust calibration

Call for Submissions

ASSURE-AI solicits original submissions that advance research and practice in security assurance for AI-assisted software engineering.

Submission Type Page Limit Scope
Research papers Up to 8 pages Original research on security assurance for AI-assisted software engineering
Short papers Up to 5 pages Early results, focused empirical studies, experience reports, and emerging ideas
Tool, dataset, and benchmark papers Up to 4 pages Tools, datasets, evaluation frameworks, or benchmarks

Submission Guidelines

All submissions must be written in English and submitted in PDF format through the ICSE 2027 workshop HotCRP system. Submissions must follow the IEEE conference proceedings format and must not exceed the page limits listed above. Submissions must be original and not under review elsewhere. Each submission will be reviewed by at least three members of the program committee and evaluated for relevance, technical quality, novelty, clarity, and potential to stimulate workshop discussion. The workshop will follow a single-anonymous review process.

Workshop Format

ASSURE-AI is planned as a one-day interactive workshop. The program will combine invited and peer-reviewed content with structured discussions to encourage community building around evidence-based security assurance for AI-assisted software engineering.

  • Invited keynote talk
  • Presentations of accepted research, position, tool, dataset, and benchmark papers
  • Panel discussion on false assurance in AI-assisted software engineering
  • Breakout discussions on evidence models, benchmarks, evaluation methods, and human review
  • Closing session to identify open research problems and collaboration opportunities
